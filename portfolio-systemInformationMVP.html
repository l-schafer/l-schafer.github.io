System Information

  • Category: Mobile Development (Android : Java)
  • Course: Mobile App Development
  • Completed: 6 May, 2022

Overview

This app was completed as part of my term project.
It allows for the user to see information about their device that would be hard or impossible to find otherwise via the stock android settings app.

It should be noted that the information panels shown are scrollable in two dimensions, but I cannot demonstrate this here, since if I were to make a screen recording on my phone it would expose potentially sensitive information. Screenshots shown are from a virtual machine.


Need

The user cannot access the majority of device metrics collected by android.
Even with developer mode enabled, the majority of this information is still hidden.
Data shown to the user is very simplified and non-technical, making it not as useful


Approach

Provide the user with access to all device data.
Present the data in a friendly and non-cluttered way.
Create a modular design to allow for compatibility with the diverse android ecosystem


Benefits

  • Easy for the user to access otherwise hidden information
  • Quick access to information without using developer settings
  • The app does not collect or sell the user’s data, and is offered free of charge.
  • Free & Open-Source Software

Design

  • Intuitive
    • Information is categorized
    • Data is acquired as-needed
    • All displayed metrics are dynamically generated
  • Accessible
    • Light and Dark theme support, adjustable in the settings menu.
    • High-contrast text

What I Learned

  • An alarming amount of system information can be acquired without requesting any permissions
  • Subclasses of android.system, and android.hardware are challenging to use, and require frequent reference to their documentation.
    • Each subclass often required a new way to parse its corresponding information, providing a challenge
  • Programmatically generating layout elements is a lot more efficient than having pre-made ones.

Future Development

  • Have each list item on the main activity be expandable, showing at-a-glance information.
  • Instantiate each category’s activity as a reusable fragment, since all the individual layouts are identical.
  • Add more categories
    • For this project, I focused on trying to get as much information as I could without requesting additional permissions
    • Bluetooth, WiFi, GPS, NFC, Device Sensors, and Camera modules
  • Add more settings for user-friendliness
    • Allow choosing reported units (GB, MB, KB), Fahrenheit/Celsius, etc.
  • Add dynamically updated information, graphs, and statistics.
